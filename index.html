Project Overview & Role

  • 역할: AI 평가 서비스 아키텍처 설계, 백엔드 시스템 개발, 운영 안정화 영역 주도
  • 서비스 특성: 음성 데이터 기반 AI 평가 및 개인화 피드백 제공 서비스
  • 주요 고려사항: LLM API 비용 관리, 평가 결과 재활용, 평가 정책 변경 대응, 운영 자동화

Architecture Decision

  • AI API 결과와 평가 Rule Engine 분리
  • LLM 분석 결과(JSON)를 저장하고 평가 Rule Engine과 분리하여 재채점 시 LLM 재호출 없이 기존 결과를 활용하는 구조 설계
  • AI API 처리 특성을 고려한 Async Processing 구조 적용
  • 운영자가 평가 정책을 변경할 수 있는 Admin 기반 운영 구조 설계

문제 정의 및 해결 전략 (Troubleshooting)

  • 해결 과제: OpenAI API 호출 비용 관리, AI 평가 결과 재활용 필요성, 평가 정책 변경에 따른 반복 처리 문제
  • 비동기 파이프라인: asyncio 기반 동시성 제어 구조를 적용하여 외부 AI API 호출 병목 완화
  • 재채점 구조 설계: 점수 계산 로직을 독립적으로 분리하여 AI API 재호출 없이 기존 데이터만으로 빠른 재채점이 가능하도록 구조 개편
  • 주요 성과: 구조 개편을 통해 재채점 과정에서 AI API 재호출 0회 달성. 비동기 처리 파이프라인으로 응답 처리 시간 단축 및 운영 개입 최소화 등 서비스 안정성과 비용 절감을 성공적으로 확보

Key Architectural Decisions

  • 도메인 서비스 최적화: 상품·행사·주문 등 주요 도메인의 서비스 분리 구조에서 담당 영역의 API 설계, 데이터 구조 개선, 조회 성능 최적화를 수행한 사례
  • Front API / Back API 계층 분리: 고객용 대용량 조회 트래픽과 관리자/파트너용 CUD 처리 트래픽을 독립적으로 스케일아웃
  • Master / Replica DB 기반 Read/Write 분리: 읽기 전용 DB 리플리카 구성을 통한 데이터베이스 읽기 부하 분산
  • 도메인 데이터 특성에 맞춘 캐시 엔진 이원화 (Redis / Couchbase): 상품(Key-Value)과 행사(Document) 도메인의 데이터 구조적 차이에 맞춰 캐시 솔루션을 분리 배치, DB I/O 병목을 근본적으로 차단하고 API 응답 속도 최적화
